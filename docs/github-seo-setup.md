# GitHub SEO Setup Checklist

Use this checklist to maximize the discoverability of the closed-source Instagram Detailed List Viewer repository on GitHub and search engines.

## Repository Name

Recommended repository name:

```text
instagram-detailed-list-viewer
```

This name includes the primary keyword phrase and clearly describes the app.

## Repository Description

Use this GitHub repo description:

```text
Closed-source Instagram data export viewer for followers, following, not-following-back reports, list comparison, and AI-powered insights.
```

## Website Field

Use the official product landing page if one exists. If not, use the GitHub Pages documentation URL after Pages is enabled.

## GitHub Pages

Enable GitHub Pages from the `docs` folder on the `main` branch. Use `docs/index.md` as the documentation home.

Suggested page title:

```text
Instagram Detailed List Viewer Documentation
```

Suggested meta description:

```text
Open Instagram data export files, inspect followers and following lists, compare snapshots, and generate not-following-back reports without Instagram login credentials.
```

## README Priorities

The README should:

- Start with the exact product name.
- Explain the app in the first two sentences.
- Include keywords naturally: Instagram data export viewer, followers list analyzer, following list viewer, not-following-back report.
- State that the app is closed source.
- State that no Instagram login is required.
- Link to focused docs pages.
- Include a disclaimer that the app is independent and not affiliated with Instagram or Meta.

## Release Strategy

Each GitHub Release should include:

- Version number
- Plain-language summary
- Installer or download link, if public
- Supported operating systems
- New features
- Fixes
- Known limitations
- Privacy notes for any AI-powered insight changes

Example release title:

```text
Instagram Detailed List Viewer v1.0.0 - Instagram export viewer with follower comparison
```

## Issue Templates

Use issue templates for:

- Bug reports
- Feature requests
- Export format compatibility requests

This helps long-tail discovery because real issues often include the exact phrases users search for.

## Link Building Notes

Link this repository from:

- Official product website
- Product Hunt listing
- AlternativeTo profile
- SaaSHub profile
- Indie Hackers launch post
- DEV.to or Hashnode tutorial
- YouTube demo description
- Reddit feedback post, where allowed by community rules

Avoid spammy backlink blasts. The strongest links come from useful pages that explain a real workflow.
